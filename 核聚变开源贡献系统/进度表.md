## 多项目并进机制 v2.1

> **治本目标**：避免「单仓 1 PR 干等 review」阻塞整个系统。允许多项目同时活跃。
> 之前：单项目单轮 → PR 提交后只能等 → 系统空转。
> 现在：每 tick 决策多项目状态 → 选最高优先级行动 → 无 review 不阻塞新项目。
> 
> **每 tick 决策矩阵**（从高到低）：
> 1. **REVISION**：任意项目收到 review/comments → 处理该反馈
> 2. **CODE**：任意项目无 PR 且未达 1/1 仓限 → 启动 P1.1 研读 / 继续 P2 开发
> 3. **MONITOR**：所有项目 PR submitted 且无 review → 巡检 + 系统维护
> 4. **STALLED**：所有项目 PR > 7d 无 review → 巡检 + 考虑主动 comment ping
> 5. **BUGFIX**：发现严重系统 bug → 优先修系统
> 
> **约束（保留）**：
> - 单仓 PR ≤ 1（rule #1）
> - 全局 PR ≤ 5
> - 错误预算耗尽 → conservative
> 
> **新约束**：
> - MAX 活跃项目数 = 3（超则优先清理 STALLED）
> - 7 天无 review 仍 PR open → STALLED（不再阻塞其他项目）

**策略自动调整规则**（任一触发则改写 STRATEGY_MODE）：
- 连续 3 个 PR 在 7 天内无 review 活动 → `aggressive` → `conservative`
- 合并率 ≥ 60% 累计 5 PR → `conservative` → `aggressive`
- 同一仓库连续 2 个 PR 被 reject → 加入黑名单 30 天
- 单 PR review 周期 > 30 天 → 改投同优先级低门槛备选
